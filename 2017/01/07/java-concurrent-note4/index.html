并发编程实战笔记——任务执行与中断取消

2017-01-07
并发, 读书笔记

线程中执行任务

  • 顺序去执行，单线程
  • 每个任务创建一个线程执行
  • 线程池执行，复用线程

无限创建线程缺点：
1、线程声明周期的开销，线程的创建和销毁会耗费一定的资源，同时创建线程会带来一定的处理延迟
2、资源消耗量，大量空闲线程会占用大量内存资源，大量线程竞争会耗费cpu资源
3、上下文切换开销，线程切换，cpu会保存现场和恢复现场，存在性能消耗
4、稳定性，需要限制线程的数量，限制会受到jvm参数、线程的栈大小以及操作系统的线程限制，主要限制因素为线程栈的地址空间，每个线程会维护两个执行栈，一个用于Java代码，一个用于原生代码，jvm默认会创建一个组合栈，大小在半兆左右。

Executor框架

Executor框架将任务提交和任务执行进行解耦

执行策略

在选择线程执行策略的需要考虑的：

  • 任务在什么线程中执行
  • 任务以什么顺序执行（FIFO, LIFO, 优先级）
  • 可以有多少个任务并发执行
  • 可以有多少个任务进入等待执行队列
  • 如果系统过载，需要放弃一个任务，应该挑选哪一个任务？如果通知应用程序
  • 在一个任务执行前后需要做什么处理
Executors工厂类
  • newFixedThreadPool：创建固定数据线程池
  • newCachedThreadPool：创建可缓存的线程池，不会对线程数目做限制
  • new SingleThreadPool：创建一个单线程，如果线程异常结束，会有另一个取代它
  • enw ScheduleThreadPool：创建一个定长线程池，支持定时以及周期性的任务执行
Executor生命周期

ExecutorService暗示生命周期存在运行、关闭和终止三个状态，当线程池进行shutdown时，会使一个平缓的关闭过程，先停止接收任务，然后等待所有任务执行完毕，转入终止状态，shutdownNow是一个强制关闭的状态，会尝试取消运行中以及队列中尚未开始的任务。

周期性任务

在处理周期性的任务的时候可以用Timer或ScheduleThreadPoolExecutor来处理，但Timer在jdk1.5以上基本没有使用，因为Timer支持的是基于绝对时间，而非相对时间，因此对系统时钟改变是很敏感的，ScheduleThreadPoolExecutor只支持相对时间。

高并行性
  • 线程池的核心线程的并发度与cpu核数相关线程数计算
  • 通过设置超时时间来确定线程等待时间，若超出等待时间则取消计算（Future.get()）
